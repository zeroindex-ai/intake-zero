Format: [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

dated milestone headings (`v0.1`, `v0.2`, …) rather than npm SemVer tags.

> Scope: a CHANGELOG records *what changed* for someone outside the code (a user,
> release notes). It is **not** the PROJECT.md decision log, which records *why*
> (architecture/reasoning, internal). Keep them separate; don't duplicate.
